Git & Git Installation: Complete Guide

1. What is Git?

Git is a free, open-source distributed version control system created by Linus Torvalds in 2005 for Linux kernel development.

Purpose: Tracks changes in any set of files. Used for coordinating work among programmers. Every Git directory on every computer is a full repository with complete history.

Key Features:

  1. Distributed: Every clone is a full backup with complete history. No single point of failure.
  2. Branching & Merging: Create isolated branches for features. Merge them back. Branching is lightweight and fast.
  3. Speed: Most operations are local. Commit, branch, merge, diff happen in milliseconds.
  4. Data Integrity: Every file and commit is checksummed with SHA-1 hash. Git detects corruption.
  5. Staging Area: git add moves changes to staging before committing. You control what goes in each commit.
  6. Offline Work: Commit, branch, view history without internet. Only push/pull need network.

2. How Git Works: Three States

State Location Description
Modified Working Directory You changed a file but haven't staged it
Staged Staging Area You ran git add. Ready for next commit
Committed .git Directory Data safely stored in local database

3. Git Installation

3.1 Windows

  1. Go to https://git-scm.com/download/win
  2. Download 64-bit Git for Windows Setup
  3. Run .exe file with these options:
    • Git Bash Here
    • Git from command line and 3rd-party software
    • Use OpenSSL library
    • Checkout Windows-style, commit Unix-style line endings
    • Use MinTTY terminal emulator
  4. Click Install → Finish

Verify installation in Git Bash:

git --version

3.2 macOS

Option 1: Xcode Command Line Tools

git --version

If not installed, macOS prompts you. Click Install.

Option 2: Homebrew

brew install git

3.3 Linux Ubuntu/Debian

sudo apt update
sudo apt install git -y
git --version

3.4 Linux Fedora

sudo dnf install git
git --version
Important: After installing, configure your name and email. This appears on every commit.

5. Key Git Terms

Term Definition
Repository Project folder tracked by Git. Contains .git folder
Commit Snapshot of your repo at a point in time with message
Branch Movable pointer to a commit. main is default branch
HEAD Pointer to your current branch/commit
Clone Download a repository to your computer
Push Upload local commits to remote
Pull Download commits from remote and merge

6. Essential Git Commands

6.1 Starting a Repository

git init                    # Create new local repo in current folder
git clone <url>             # Copy existing repo from URL

6.2 Basic Snapshotting

git status                  # Show changed, staged, untracked files
git add filename.txt        # Stage one file
git add .                   # Stage all changed files
git commit -m "Message"     # Save staged changes with message
git diff                    # Show unstaged changes
git diff --staged           # Show staged changes

6.3 Branching & Merging

git branch                  # List all branches. * = current
git branch feature          # Create new branch named feature
git checkout feature        # Switch to feature branch
git checkout -b bugfix      # Create and switch in one command
git merge feature           # Merge feature into current branch
git branch -d feature       # Delete branch after merge

6.4 Inspecting History

git log                     # Show commit history
git log --oneline           # Compact history: one line per commit
git log --graph             # Show branch/merge graph

6.5 Remote Repositories

git remote add origin <url> # Link local repo to remote
git remote -v               # Check remote URL
git push -u origin main     # First push to remote
git push                    # Push new commits
git pull                    # Download + merge from remote

7. Standard Git Workflow

# 1. Start new project
mkdir my-project
cd my-project
git init

# 2. Create file and commit
echo "Hello Git" > README.md
git add README.md
git commit -m "Initial commit"

# 3. Make changes
echo "Line 2" >> README.md
git status
git diff
git add README.md
git commit -m "Add line 2"

# 4. View history
git log --oneline

8. .gitignore File

Tells Git which files/folders to ignore. Create file named .gitignore:

# Dependencies
node_modules/

# Environment variables
.env

# Log files
*.log

# OS files
.DS_Store
Thumbs.db

# Build outputs
build/
dist/

# IDE folders
.vscode/
.idea/

9. Why Developers Use Git

  1. Track History: Revert to any previous version if code breaks
  2. Experiment Safely: Try features in branches without affecting main code
  3. Collaborate: Multiple people work on same files without overwriting
  4. Backup: Every clone is full backup of entire project history
  5. Blame/Praise: git blame shows who changed each line and why
  6. Industry Standard: Required for 95% of software jobs

10. Summary

  1. Git = distributed version control system on your computer
  2. Install from git-scm.com for your OS
  3. Configure name/email with git config --global
  4. Basic cycle: git add → git commit → git status → git log
  5. Use branches to isolate work
  6. .gitignore prevents tracking unwanted files
